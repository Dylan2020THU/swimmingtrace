# #4d API Keys 实现计划（TDD）

spec：`docs/superpowers/specs/2026-06-30-api-keys-design.md`。分支 `feat/api-keys`。

## Task 1 — shared 类型 + billing features
- shared：`ApiKeyListItem`、`CreatedApiKey`、`CreateApiKeyDto`；`PlanInfo.features` 加 `apiKeys: boolean`。
- billing `plan.config.ts`：features 加 `apiKeys`（FREE false / PRO true）；`PlanLimits.features` 类型加 `apiKeys`。
- `BillingService.assertFeature` 联合类型加 `'apiKeys'` + label「API Keys」。
- `npm run build -w @swim/shared`。

## Task 2 — Prisma 模型 + 迁移
- `schema.prisma`：`ApiKey` 模型 + `User.apiKeys ApiKey[]`。
- 手写迁移 `20260630010000_add_api_key`：CREATE TABLE + unique(keyHash) + index(ownerId) + FK cascade。
- `prisma generate` + dev `migrate deploy`。

## Task 3 — api-key util（红→绿）
- `api-keys/api-key.util.ts`：`API_KEY_PREFIX`、`generateApiKey`、`hashApiKey`。
- `api-key.util.spec.ts`：plaintext 以 `swk_` 开头；prefix=前 12 且为 plaintext 前缀；hashApiKey 确定性、与明文不同；两次生成不同。

## Task 4 — ApiKeysService（红→绿）+ Module
- `api-keys/api-keys.service.ts`：create/list/revoke（注入 Prisma + BillingService）。
- `api-keys.service.spec.ts`（mock prisma + billing）：create 调 assertFeature、返回明文、`prisma.apiKey.create` data 含 keyHash 不含明文；list 映射无密字段；revoke 用 `deleteMany({where:{id,ownerId}})`（归属即条件）。
- `api-keys.module.ts`：imports [BillingModule]，providers [ApiKeysService, PrismaService]，controllers [ApiKeysController]。

## Task 5 — 增强 guard + Controller + 接线
- `common/auth.common.ts`：`JwtAuthGuard` 注入 `PrismaService`，`canActivate` 加 `swk_` 分支（util hashApiKey 查库 → 设 owner + 刷 lastUsedAt；失效 401；否则 super）。
- `auth.common.spec.ts`（或新建）：mock prisma + context，验证 key 命中设 user、失效 throw、非 key 调 super（可 stub）。
- `api-keys.controller.ts`：`@Roles(OWNER)` 三端点。
- `app.module.ts`：imports ApiKeysModule。
- **校验所有用 JwtAuthGuard 的模块都提供 PrismaService**（否则启动 DI 失败）——逐一确认/补。
- 跑 api 单测全绿。

## Task 6 — e2e
- `test/api-keys.e2e-spec.ts`：Pro owner 建 key→明文→`Bearer swk_` 调 `POST /pools`/`GET /pools` 成功→list 见 prefix+lastUsedAt→DELETE→该 key 再调→401；Free owner 建 key→402；坏 `swk_` →401。
- beforeAll 清库（含 apiKey、idempotencyKey）。
- 跑 e2e 全绿。

## Task 7 — 前端
- web endpoints：`listApiKeys`/`createApiKey`/`revokeApiKey`。
- `features/account/ApiKeysCard.tsx`：列表 + 新建弹窗（明文一次 + 复制 + 警示）+ 撤销；按 `usePlan().features.apiKeys` 门禁 UI。
- `AccountPage` 接入；web msw 默认 `/api/api-keys` GET handler。
- `ApiKeysCard.test.tsx`：列表渲染、新建显示明文、撤销调 DELETE。
- 跑 web 单测全绿。
